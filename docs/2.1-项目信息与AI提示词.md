## 三、项目全流程 AI 大模型提示词

### 3.1 需求分析阶段

```
你是一个软件工程需求分析师。请为"光影手记 (Shimmer)"——一款基于 AI 的智能照片管理与分享平台，
梳理完整的需求分析文档，包括：
1. 用户角色（访客、注册用户、管理员）
2. 各角色的功能需求
3. 非功能需求（性能、安全、可用性）
4. 核心业务流程图

平台核心功能包括：
- 照片上传与管理（单张/批量、缩略图自动生成、EXIF 提取）
- AI 智能辅助（自动生成标题/心情/标签、搜索增强、年度回顾叙事、故事线摘要）
- 标签系统（自定义标签、AI 建议标签一键添加）
- 相册管理
- 收藏功能
- 时间轴/故事线浏览
- 年度回顾与统计
- 分享卡片生成
- 用户系统（JWT 认证、权限控制）
```

### 3.2 概要设计阶段

```
请为"光影手记"项目设计系统架构，包含：
1. 系统分层架构图（展示前端、后端、数据库、对象存储、AI 服务的关系）
2. 数据库 ER 图，包含以下表：users, photos, tags, photo_tags, favorites, albums, album_photos, ai_settings, share_cards, review_cache, story_summaries
3. 功能模块划分
4. 前后端接口设计规范（RESTful API）

技术约束：
- 后端使用 Express.js + MySQL
- 前端使用 Vue 3 + Vite + Pinia
- 图片存储在 Cloudflare R2
- AI 支持 OpenAI/Ollama/智谱三种 provider
```

### 3.3 服务端编码阶段

```
请使用 Express.js 框架实现以下 API 路由，要求使用 ES Module 语法：

1. 认证模块 (/api/auth)：注册、登录、获取当前用户、修改密码
   - JWT 令牌认证，bcrypt 密码加密
2. 照片模块 (/api/photos)：CRUD、列表查询（支持排序/年份/月份/搜索/标签筛选）
   - 可见性控制（公开/私密）
   - 管理员可查看全部照片
3. 上传模块 (/api/upload)：单张/批量上传
   - Multer 接收文件，Sharp 压缩为 1920px 宽度并生成 400px 缩略图
   - 上传至 Cloudflare R2，提取 EXIF 数据
4. 标签模块 (/api/tags)：标签 CRUD、照片-标签关联
5. AI 模块 (/api/ai)：AI 配置管理、元数据生成、搜索重写
6. 中间件：authenticateToken, optionalAuth, requireAdmin, requireAuth, errorHandler

代码规范：
- 统一使用 async/await 错误处理
- 所有数据库操作使用 mysql2/promise 连接池
- 统一 JSON 响应格式
```

### 3.4 客户端编码阶段

```
请使用 Vue 3 Composition API + Vite 实现以下页面：

1. 首页 (Home.vue)：浮动卡片风格照片瀑布流，支持搜索和标签筛选
2. 时间轴 (Timeline.vue)：按年份/月份分组的照片浏览
3. 管理后台 (Admin.vue)：照片上传与管理，AI 自动补全按钮
4. 暗房 (Darkroom.vue)：私密照片空间，包含四种模式
5. 年度回顾 (Review.vue)：年度统计数据与 AI 叙事
6. 故事线 (Storyline.vue)：按月聚合的故事列表
7. 分享卡片 (ShareCard.vue)：四种模板的分享卡片生成
8. AI 设置 (Settings.vue)：AI 服务提供商/模型/API Key 配置

技术要求：
- Pinia 管理登录状态
- Axios 封装 API 请求，请求/响应拦截器
- Vue Router 路由守卫控制访问权限
- 响应式设计，支持暗色模式
```

### 3.5 AI 功能集成阶段

```
为"光影手记"实现 AI 功能集成，需要支持多种 AI 服务提供商：

1. 系统提示词（System Prompt）设计：
   - 照片元数据生成：分析照片内容生成标题、心情和标签
   - 年度回顾叙事：根据年度统计数据撰写有温度的回顾文字
   - 搜索建议：将用户搜索词扩展为关键词和标签
   - 故事线摘要：为一组同一天同一地点的照片生成叙事摘要

2. AI Provider 适配：
   - OpenAI 兼容接口：使用 /v1/chat/completions
   - Ollama 本地模型：支持多模态视觉模型
   - 智谱 AI：使用 /chat/completions（不带 /v1 前缀）

3. 错误处理：
   - API Key 无效、模型不存在、超时、网络错误、频率限制
   - 每种错误返回对应用户友好的中文提示

4. 缓存策略：
   - 年度回顾 AI 结果存入 review_cache 表
   - 故事线 AI 摘要存入 story_summaries 表
   - 支持手动重新生成
```
