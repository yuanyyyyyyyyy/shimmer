## 二、中间件链与路由设计

### 2.1 中间件链

```plantuml
@startuml
skinparam backgroundColor #FEFEFE

title HTTP 请求中间件处理链

actor Client as C
participant "CORS" as COR
participant "JSON解析\nbody-parser" as JSON
participant "请求日志" as LOG
participant "认证中间件" as AUTH
participant "路由处理" as ROUTE
participant "错误处理" as ERR

== 公共请求（无需登录） ==
C -> COR: GET /api/photos
COR -> JSON: 通过
JSON -> LOG: 解析完成
LOG -> ROUTE: GET /api/photos (列表)
ROUTE --> C: 200 JSON

== 需认证请求 ==
C -> COR: POST /api/photos (需登录)
COR -> JSON: 通过
JSON -> LOG: 解析完成
LOG -> AUTH: 提取 Bearer Token
AUTH -> AUTH: JWT 验证\n数据库查询用户
AUTH -> ROUTE: req.user 挂载
ROUTE --> C: 200 JSON

== 错误场景 ==
C -> AUTH: 无效/过期 Token
AUTH --> ERR: 401/403
ERR --> C: 错误 JSON

== 404 场景 ==
C -> ROUTE: 不存在的路径
ROUTE --> ERR: notFoundHandler
ERR --> C: 404 JSON

@enduml
```

### 3.4 照片上传与处理 (routes/upload.js)

```plantuml
@startuml
title 照片上传时序图

actor 用户 as C
participant "前端" as FE
participant "POST /api/upload" as API
participant "Multer" as MUL
participant "Sharp\n图片处理" as SHP
participant "R2 对象存储" as R2

C -> FE: 选择图片文件
FE -> API: POST /api/upload (multipart)
API -> MUL: upload.single('photo')

MUL --> API: req.file (Buffer)

API -> SHP: processImage(buffer)

SHP -> SHP: 提取 EXIF 数据\n(相机/镜头/光圈/快门/ISO/焦距)
SHP --> API: { url, thumbnailUrl, width, height, file_size, camera, lens, ... }

API --> FE: 200 { message, url, thumbnailUrl, ...exif }
FE -> C: 显示上传成功

@enduml
```

### 3.6 认证流程

```plantuml
@startuml
title 用户认证流程 (JWT)

actor 用户 as C
participant "前端" as FE
participant "POST /api/auth/login" as LOGIN
participant "bcrypt" as BCR
participant "jwt" as JWT
database "MySQL" as DB

== 注册 ==
C -> FE: 填写用户名/密码
FE -> LOGIN: POST /auth/register
LOGIN -> DB: 检查用户名是否已存在
DB --> LOGIN: 结果
LOGIN -> BCR: bcrypt.hash(password, saltRounds=10)
BCR --> LOGIN: password_hash
LOGIN -> DB: INSERT INTO users (username, password_hash, ...)
DB --> LOGIN: user
LOGIN -> JWT: jwt.sign({ userId, username, role }, JWT_SECRET, { expiresIn: '7d' })
JWT --> LOGIN: token
LOGIN --> FE: 200 { token, user }
FE -> FE: 存储 token 到 localStorage

== 后续请求 ==
FE -> 任意API: Authorization: Bearer <token>
API -> API: jwt.verify(token, JWT_SECRET)
API -> DB: SELECT user FROM users WHERE id = ?
DB --> API: user
API -> API: req.user = user, next()
API --> FE: 200 业务数据

@enduml
```

### 3.7 错误处理中间件 (middleware/error.js)

错误处理中间件采用自定义错误类（`AppError`、`ValidationError`）与全局错误处理器相结合的方式，统一捕获和格式化 API 错误响应。核心逻辑包括：区分可操作错误与内部错误、处理 MySQL 重复键/外键异常、生产环境隐藏错误细节。
