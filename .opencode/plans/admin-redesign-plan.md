# Admin.vue 照片管理页面重构计划

## 一、配色统一

### 1.1 删除 Admin 内部变量，复用全局变量

**删除的变量（7 个）：**
- `--admin-primary` → 改用 `--text-color`
- `--admin-secondary` → 移除（无对应用途）
- `--admin-text` → 改用 `--text-color`
- `--admin-text-secondary` → 改用 `--text-secondary`
- `--admin-text-tertiary` → 改用 `--text-tertiary`
- `--admin-bg` → 改用 `--bg-color`
- `--admin-card` → 改用 `--card-bg`
- `--admin-hover` → 改用 `--hover-bg`
- `--admin-border` → 改用 `--n-300`（App.vue 已定义）
- `--admin-border-light` → 改用 `--n-200`
- `--admin-accent-danger` → 保留为 `--danger-color`（新增到全局或组件内局部定义）

**对应的 `.dark` 模式块直接删除**，全局变量自动处理深色适配。

### 1.2 颜色替换映射表

| 旧引用 | 替换为 | 说明 |
|--------|--------|------|
| `var(--admin-primary)` | `var(--text-color)` | 主文字色 |
| `var(--admin-secondary)` | `var(--secondary-color)` | 主题蓝 |
| `var(--admin-bg)` | `var(--bg-color)` | 页面背景 |
| `var(--admin-card)` | `var(--card-bg)` | 卡片背景 |
| `var(--admin-hover)` | `var(--hover-bg)` | 悬停背景 |
| `var(--admin-border)` | `var(--n-300)` | 边框色 |
| `var(--admin-border-light)` | `var(--n-200)` | 浅边框 |
| `var(--admin-text-secondary)` | `var(--text-secondary)` | 次要文字 |
| `var(--admin-text-tertiary)` | `var(--text-tertiary)` | 辅助文字 |
| `#c0392b` / `#e74c3c` | `var(--danger-color)` | 危险操作（统一用变量） |

## 二、按钮重构

### 2.1 页面顶部按钮组（page-actions）

**当前状态：** 标题与副标题改为同行显示，三个 pill 按钮横排 — 「上传照片」+ 「AI 设置」(icon) + 「登出」(icon)

**重构方案：**
- 标题改为 `.page-header` 风格，与 Albums.vue 一致
- 副标题与标题同行（`align-items: baseline`），减少垂直空间
- 按钮组使用 `.page-actions` 容器，三个 pill 按钮统一风格

```
┌─────────────────────────────────────┐
│  管理后台 · 11 张照片 · 1774 年记录   │
│                      [+ 上传照片] [⚙] [⏻] │
└─────────────────────────────────────┘
```

### 2.2 照片列表操作按钮（photo-actions）

**当前状态：** 每行右侧有编辑图标 + 删除图标，单独放置

**重构方案：**
- 保持图标按钮，但统一样式：圆角 + hover 效果
- 「可见性切换」从独立 badge 改为可点击的 toggle 按钮
- 操作按钮组：`[可见性] [编辑] [删除]` 水平排列

```
┌──────────────────────────────────────────────────┐
│  [缩略图]  标题/心情/元信息        [公开✓] [✎] [🗑]  │
└──────────────────────────────────────────────────┘
```

### 2.3 弹窗内按钮（modal-form）

**当前状态：**
- 「取消」和「保存」在表单底部
- 「AI 自动补全」按钮 + 提示文字在表单中间
- 「新建标签」在标签选择区末尾

**重构方案：**

**底部操作栏：**
```
┌─────────────────────────────────────────┐
│  [取消]              [保存] / [保存 N 张]  │
└─────────────────────────────────────────┘
```
- 左侧：取消（`btn-secondary`，轻量样式）
- 右侧：保存（`btn-primary`，实心高亮）

**AI 操作区：**
- 「AI 自动补全」按钮 + 加载状态文字，紧凑一行
- AI 建议标签紧跟其下，形成独立区块（带浅色背景）

## 三、弹窗布局优化（保持上下结构）

### 3.1 整体结构调整

**当前：** 预览区 + 表单区 无明显分隔
**优化：** 用视觉分层区分预览区和表单区

```
┌──────────────────────────────────────┐
│            照片预览区域                │
│  ┌────────────────────────────────┐  │
│  │         图片 / 占位符            │  │
│  └────────────────────────────────┘  │
│                                      │
│  ────────── 或分隔线 ──────────────  │
│                                      │
│  📝 基本信息                          │
│  [标题] [心情]                        │
│                                      │
│  🤖 AI 助手                          │
│  [AI 自动补全]                        │
│  [标签1] [标签2] [标签3]              │
│                                      │
│  📷 拍摄信息                          │
│  [日期] [地点] [相机信息]              │
│                                      │
│  🔒 可见性                            │
│  (公开) (私密)                        │
│                                      │
│  🏷️ 标签                             │
│  [标签1] [标签2] [+ 新建标签]          │
│                                      │
│  ──────────────────────────────────  │
│  [取消]                    [保存]     │
└──────────────────────────────────────┘
```

### 3.2 具体优化点

1. **预览区域**：保持当前结构，但增加 `border-bottom` 或 `margin-bottom` 分隔
2. **表单分组**：用 section 标题 + 分组框将表单分为：
   - 基本信息（标题、心情）
   - AI 助手（按钮 + AI 标签）
   - 拍摄信息（日期、地点、EXIF）
   - 可见性设置
   - 标签选择
3. **间距统一**：section 之间 `24px`，组内 `16px`
4. **AI 区域**：浅色背景（`var(--n-200)`），与其他表单区分
5. **标签选择**：增加 "管理标签" 入口或提示
